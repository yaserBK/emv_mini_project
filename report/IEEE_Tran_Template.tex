\section{Introduction}

Automated quality control on production lines demands inspection systems that are fast, consistent, and cost-effective. These requirements lend themselves to the use of embedded machine vision specifically, as it can be built as an add-on to the existing production line, is often cheap on hardware (most hardware for embedded machine vision is under €1000), operates at low power, and utilises cameras capable of dealing with several items per second.

This project addresses the detection of defective date stamps on milk bottle caps, a task where defects (smeared ink, missing digits, print head misalignment, etc.) are rare, often visually subtle, and difficult to gather a large dataset of obvious failures. Additionally, these failures must be caught at production time, before the product is packed for shipping, otherwise the product will be wasted due to customer rejection upon delivery.

The project proposes a system for single-class anomaly detection. Rather than training a binary classifier that requires a balanced dataset of correct and incorrect sample images, the model learns a statistical description of acceptable date stamps only. Any cap whose date stamp features deviate from this learned distribution is then flagged as anomalous. This is well-suited to real-world manufacturing, where defect samples are often scarce and defect modes may not even be known in advance.

The pipeline combines a frozen, pre-trained convolutional image recognition network
(ResNet-18~\cite{he2016resnet}) which converts images into numerical feature vectors, with a multivariable Gaussian distribution which models acceptable bottle caps. The model flags anomalies using Mahalanobis distance scoring to compare how far a given sample sits from the distribution against a threshold derived from the distribution. Both inference and distribution training run in real-time on an NVIDIA Jetson Nano, low-power system-on-module development board which is representative of hardware available for industrial embedded machine vision. The key design goals were:
\begin{itemize}
\item High anomaly recall with low false-alarm rate
\item Real-time throughput (i.e. $\geq$30\,FPS)
\item Low-power system theoretically suited to production deployment (i.e. 5-10w power budget)
\end{itemize}

\section{Background Research}
\subsection{Dataset}
As existing datasets did not provide adequate coverage of the font, font size, and date stamp shape of the particular production line targetted by this project, a custom dataset was built by capturing images of bottle caps coming from the production line and preprocessing to give consistent size, crop, and orientation to the images in the dataset. The images were then sent through an image processing pipeline which isolates the bottle cap from the image, at which point the images were passed to the distribution training script. The dataset consists of 1,025 augmented 256x256 crops of known good bottle caps, derived from the preprocessed images and expanded with photometric and geometric augmentations (gamma, saturation, brightness, and Gaussian noise).

Validation uses 395 known good and 57 intentionally defective crops. Because one-class learning only requires normal examples during training, collecting labelled anomalies is needed only for validation, which reduced the data acquisition burden significantly, which is a common motivation for anomaly-detection approaches in modern industrial inspection~\cite{bergmann2019mvtec}.

\subsection{Feature Extraction}
Collecting sufficient data to train a deep network from scratch is impractical for this purpose as previously outlined. Transfer learning from the ImageNet-pretrained~\cite{imagenet} ResNet-18 model covers a sufficient amount of visual features without domain-specific supervised training. The final fully-connected layer (the head) is removed from the model, exposing the penultimate layer (a global average pooling layer) output: A 512-dimension feature vector describing the input image. The model backbone is frozen throughout, so no GPU memory is consumed by gradients and inference is very lightweight as the model does not back propagate. This is important for Jetson
deployment where memory and GPU power are high for an embedded system, but not high enough to support back propagation and real-time inference at the same time. ResNet-18 has been shown to produce strong one-class baselines in prior work~\cite{roth2022towards,defard2021padim}, and also has seen extensive use on the Jetson Nano in open-source projects, so it was an obvious choice for this project. Producing 512 features for each image, it provides an extremely detailed description of each image, which can then be used to build an accurate distribution of known good bottle cap prints.

\subsection{PCA Dimensionality Reduction}

Not all 512 ResNet features are informative for bottle-cap stamps. A pure-NumPy incremental PCA projection is applied to the dataset after running it through ResNet but prior to building the distribution. Sufficient principal components are kept to explain 95\% of training-set variance by default. This improves the covariance estimate that follows this step and also reduces inference compute which raises FPS. This ultimately boosts the accuracy of the system by discarding data which represents noise both in training and inference. This is consistent with the analysis of Defard~et~al.~\cite{defard2021padim}'s research into Patch Distribution Modelling, an alternative approach to the same methods used in this project.

\subsection{Ledoit-Wolf Gaussian Model}
The training features are modelled as a multivariable Gaussian distribution $\mathcal{N}(\boldsymbol{\mu}, \boldsymbol{\Sigma})$, where $\boldsymbol{\mu}$ is the mean feature vector of the training samples and $\boldsymbol{\Sigma}$ is the covariance matrix describing how
much the features vary and correlate with one another. The covariance matrix acts as a multivariable standard deviation.

Reliably estimating $\boldsymbol{\Sigma}$ requires many multiples more samples than there are dimensions. With a 512-dimension feature vector coming from ResNet, and a limited training dataset, the covariance matrix $\Sigma$ becomes ``ill-conditioned'', where its inverse becomes numerically unstable and fluctuates wildly with small changes to $\Sigma$. The inverse of $\Sigma$ is required for calculating the Mahalanobis distance, which is how the whole system compares input samples to the distribution. A system needs to be implemented to resolve the numerical instability without increasing the size of the training dataset by several orders of magnitude.

Ledoit--Wolf shrinkage~\cite{ledoit2004well} addresses this by blending $\mathbf{S}$, the ill-conditioned covariance matrix, with a scaled identity matrix:
\begin{equation}
    \hat{\boldsymbol{\Sigma}} = (1-\alpha)\mathbf{S} +
    \alpha\,\frac{\operatorname{tr}(\mathbf{S})}{d}\,\mathbf{I}
\end{equation}
Here $\mathbf{S}$ is the raw covariance matrix, $d$ is the feature dimensionality, $\mathbf{I}$ is the identity matrix, and $\frac{\operatorname{tr}(\mathbf{S})}{d}$ scales the identity to match the average variance of the data. $\operatorname{tr}$ is the ``trace'' of the matrix, which is the sum of values along its diagonal. The blend weight $\alpha\in[0,1]$ controls how much the estimate is pulled toward this simpler, stable structure: $\alpha=0$ gives the raw sample covariance while $\alpha=1$ ignores the data entirely. 

The resulting ``shrunken'' covariance matrix $\hat{\Sigma}$ isn't necessarily correct but \emph{is} stable, in exchange for being slightly biased towards the identity matrix, which is acceptable because the entries which are not along the diagonal of the covariance matrix are typically the ones which are least reliable.

The Ledoit--Wolf method computes the optimal $\alpha$ analytically. Their paper shows that even though the true covariance is unknown, the error between $\hat{\Sigma}$ and the true covariance matrix can still be estimated analytically, and the optimal value of $\alpha$ can be calculated from these values only. This eliminates the need for cross-validation as is typically seen in neural network training, which would require a ridiculous number of validations due to the number of dimensions and the comparatively small size of the dataset, increasing training time by several orders of magnitude. This is the same regularisation strategy used in PaDiM~\cite{defard2021padim} and related embedding-based methods of anomaly detection, specifically to avoid that issue.

\subsection{Mahalanobis Distance Scoring}
At inference, each cropped and preprocessed image is scored by its Mahalanobis distance to the training data distribution:
\begin{equation}
    d(\mathbf{x}) = \sqrt{(\mathbf{x}-\boldsymbol{\mu})^\top \cdot
    \hat{\boldsymbol{\Sigma}}^{-1}(\mathbf{x}-\boldsymbol{\mu})}
\end{equation}
where $\mathbf{x}$ is the feature vector of the image being tested,
$\boldsymbol{\mu}$ is the mean feature vector of the known-good training
samples, and $\hat{\boldsymbol{\Sigma}}^{-1}$ is the inverse of the
regularised covariance matrix.

Mahalanobis distance is scale and correlation invariant. When the distribution being measured against is Gaussian, the Mahalanobis distance has a chi-squared distribution. This means interpretable percentile-based thresholds can be defined (e.g. $p_{90}$, $p_{95}$, $p_{99}$) which correspond to the probability that a good sample would have a distance greater than the Nth percentile of the training dataset. The approach is very much classical, dating back to 1936~\cite{mahalanobis1936generalised}, but it has proven more than suitable for dealing with single-class inspection problems like anomaly detection when dealing with large feature vectors from deep neural networks~\cite{lee2018simple,roth2022towards}.

\subsection{Requirements Assessment}

The system meets the primary embedded requirements: inference operates on a frozen model and precomputed distribution, requiring no on-device training. Training can be conducted on the embedded device if needed, utilising the Jetson Nano GPU. The real-time video inference pipeline runs at $\geq$20\,FPS on the Jetson Nano hardware, utilising the GPU for ResNet-18 inference and running at an appropriate frame rate for use on a bottling line (max throughput 5 bottles/min). The percentile-based thresholding mechanism allows the sensitivity to be adjusted without retraining which is appropriate for a system of this type. Additionally, the power budget is more than respected, maxing out at 10w under normal operating conditions. Finally, the system has perfect recall. While the False Positive Rate is higher than desired, the rate is still low enough that it can be handled by the machine operator.

\section{Conclusion}
This project demonstrates that a frozen pre-trained CNN model, combined with an appropriate one-class statistical model (e.g Mahalanobis scoring, and Ledoit-Wolf covariance regularisation), results in a practical and production-deployable anomaly detection system, in this case for industrial date stamp inspection. The approach requires only ``normal'' samples for training, and is tolerant of small datasets through both covariance regularisation and dataset augmentation. The system achieves real-time inference within the power
and memory budget of the NVIDIA Jetson Nano at an appropriate frame rate for this application.

The main limitations include sensitivity to distribution shift (e.g. significant changes in lighting), and poor recall for highly localised defects (e.g. a single missing digit) that produce only a small shift in the average-pooling layer of the CNN.
